# Shree Stone Command

Inventory & invoicing app for Shree Stone Craft — Vite + React + TypeScript + Supabase.

## Stack

- **Frontend:** Vite, React 18, TypeScript, Tailwind, shadcn/ui, TanStack Query
- **Backend:** Supabase (Postgres, Auth, Storage)
- **Auth:** Supabase Auth (email/password) with `owner` / `staff` roles
- **PDF:** jsPDF + jspdf-autotable

## Project structure

```
src/
  lib/
    supabase.ts          Supabase browser client (anon key)
    auth.ts              login/logout, useAuth hook with profile + role
    store.ts             Async data layer (stones, locations, logs, invoices, image upload)
    hooks/               TanStack Query hooks
    invoicePdf.ts        PDF generation
  pages/                 One file per route
  components/            UI primitives + ProtectedRoute

supabase/
  migrations/0001_init.sql   Full schema, RLS, RPCs, storage policies
  seed/Inventory.xlsx        Source inventory file

scripts/
  import-inventory.ts        One-shot import from xlsx → Supabase

.env.local                   Your secrets (gitignored)
.env.example                 Template
```

## First-time setup

```bash
# 1. Install
npm install

# 2. Copy env template and fill in Supabase keys
cp .env.example .env.local
# Edit .env.local with VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY

# 3. Apply schema in Supabase SQL Editor
# Copy supabase/migrations/0001_init.sql → paste into SQL Editor → Run

# 4. Import inventory data (creates owner user too)
npm run import:inventory

# 5. Run dev server
npm run dev
# → http://localhost:8080
```

## Adding a new staff member

1. Owner opens **Settings → Team Members → Open Supabase Auth Dashboard**
2. In Supabase Dashboard: **Authentication → Users → Add user → Send invitation** (or set a password)
3. New user appears in Settings; owner sets role to `staff` (default) or `owner`

Roles:
- **owner** — everything + manage team members
- **staff** — full app access (inventory, invoices), but cannot manage users

## Production deployment

### Vercel (recommended)

1. Push repo to GitHub
2. Import on Vercel
3. Set environment variables:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   - (do NOT set `SUPABASE_SERVICE_ROLE_KEY` in production — it's import-only)
4. Build command: `npm run build` (default)
5. Output: `dist`

### Supabase production checklist

- [ ] Disable public sign-ups: **Auth → Providers → Email → "Enable signups" = OFF**
- [ ] Verify daily backups are on: **Database → Backups**
- [ ] Set up custom domain SMTP for password reset emails (Auth → SMTP)
- [ ] Restrict allowed redirect URLs to your production domain
- [ ] **Rotate the `service_role` key** (Settings → API → "Reset service role")

## Database schema

Generated by `supabase/migrations/0001_init.sql`:

- `profiles` — mirrors `auth.users`, holds role
- `locations` — Dukaan 1st/2nd/3rd/5th Line, Godown
- `stones` — inventory items (FK → locations)
- `stock_logs` — append-only audit trail
- `invoices` + `invoice_line_items` — invoicing with atomic `create_invoice` RPC that deducts stock + logs in one transaction
- Storage bucket `stone-images` for stone photos

All tables have RLS enabled. Authenticated users have full read/write on stones, locations, invoices, logs. Only owners can change profile roles.
